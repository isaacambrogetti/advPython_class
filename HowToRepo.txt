Guide on how to use the cloned repository without breaking it (hopefully)

> ALWAYS WORK ON BRANCHES (This Is Critical)

Case 1 — You Already Have the Branch Locally
- Check where you are:
    git branch
        If you see:     * trybranch     Then you are already on it.

- Step 1 — Make sure you're up to date
    git pull origin trybranch

- Step 2 — Make changes
    Edit files.

- Step 3 — Commit
    git add .
    git commit -m "write work description"

- Step 4 — Push
    git push
That updates the existing branch on GitHub. If there’s a Pull Request open, it updates automatically.


Case 2 — The Branch Exists on GitHub But Not Locally

- First fetch all remote branches:
    git fetch origin

- Then check available remote branches:
    git branch -r
        You’ll see something like:      
            origin/master
            origin/trybranch

- Now create a local tracking branch:
    git checkout -b trybranch origin/trybranch
 Now it is linked.

- Then follow the normal workflow:
    git add .
    git commit -m "write work description"
    git push



Case 3 - Creating a new branch

git checkout master
git pull origin master
git checkout -b branch/my-feature-name

Then:

git add .
git commit -m "Implement X"
git push -u origin branch/my-feature-name

( -u sets the branch to the one to as standard )

! If Your Coworker Pushed New Changes !
Before committing new work, always do:
    git pull origin trybranch
This prevents conflicts and divergence.

## WHEN STARTING TO WORK TO NEW HOMEWORK ##

STEP BY STEP
1. Every time before starting a new homework we merge the previous hw branch and delete it both locally and on github.
2. Then pull the changes the professors made on gitlab to the master in our github repo.
3. Then we create a new branch to work on the new hw.

Here's the steps:
1. 
git checkout master         # move to the master branch (where you want to merge the changes)
git merge trybranch         # merge the branch to the main
git branch -d trybranch     # delete the branch if it has been merged correctly it should raise any error
git branch                  # check if it has been deleted locally
git push                    # push the merge to the github repo
git push origin --delete trybranch      # delete the branch from github too

2. 
git fetch upstream                              # check the connection to the GitLab page
git log --oneline master..upstream/master       # check if there are new commits
git checkout master                             # move to master branch
git merge upstream/master                       # merges the repo to the gitlab one

if you're still working on the previous hw, do get back to that branch in this way before making changes:
git checkout my-branch                          # to go back to the branch you're working with

3.
git checkout -b branch/my-feature-name          # creates a new branch and moves it to it
git branch                                      # check if the branch is created and you're in it
